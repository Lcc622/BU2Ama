# 亚马逊跟卖自动上新 SOP

## 一、目标（Goal）

在新版本 SKU 入库后，快速完成跟卖上新，实现：

1. 新老版本同时存在购物车
2. 新版本价格低于老版本 $0.1，优先出单
3. 用真实销售测试退货率变化
4. 降低人工制表错误
5. 实现半自动化批量生成上传模板

## 二、整体流程结构

```
输入新版本 SKU
    ↓
匹配老版本 SKU
    ↓
抓取老版本 ASIN
    ↓
生成跟卖上新表
    ↓
人工复核
    ↓
上传 Amazon
```

## 三、详细操作步骤

### Step 1：输入触发

#### 目标
确认需要跟卖的新版本 SKU

- `-PH+` 后缀统一变成 `-PH`
- `-/-US+` 后缀统一变成 `-USA`

#### 操作
- 输入：**新版本 SKU（支持批量）**
- 必须为已入库 SKU
- 精确到 SKU 级别（不是 style）

---

### Step 2：匹配老版本 SKU

#### 目标
找到新版本对应的老版本 SKU

#### 逻辑
- 系统根据新版本 SKU
- 在 BI 入库更新表中匹配老版本 SKU

#### 输出：
- 新版本 SKU
- 对应老版本 SKU

> 若 BI 无法稳定匹配，可使用人工维护的新老 SKU 对照表。

---

### Step 3：抓取老版本 ASIN

#### 目标
获取老版本对应的 ASIN

#### 逻辑
- 根据老版本 SKU
- 调用 Amazon 后台 Only Thing Report 或内部接口
- 抓取老版本 ASIN

#### 输出：
- 老版本 ASIN

---

### Step 4：生成跟卖上新表（字段规则最终版）

#### 一、必须修改字段清单

以下字段为强制更新字段：

##### 1️⃣ item SKU
- **规则**：使用新版本 SKU
- **来源**：用户输入

##### 2️⃣ product ID
- **规则**：使用老版本 ASIN
- **来源**：Step 3 抓取结果

##### 3️⃣ product ID type
- **规则**：固定填写 `ASIN`
- **来源**：系统自动填充

##### 4️⃣ model style number
- **规则**：使用新版本 SKU
- **来源**：用户输入

##### 5️⃣ part number
- **规则**：使用新版本 SKU
- **来源**：用户输入

##### 6️⃣ price（standard price）
- **规则**：老版本价格 - $0.1
- **来源**：从老版本数据计算
- **公式**：`new_price = old_price - 0.1`

##### 7️⃣ list price
- **规则**：新版本价格 + $10
- **来源**：从新价格计算
- **公式**：`list_price = new_price + 10`

##### 8️⃣ quantity
- **规则**：固定填写 `0`
- **来源**：系统自动填充
- **说明**：跟卖上新时库存设为 0，避免立即出单

##### 9️⃣ image
- **规则**：清空所有图片字段
- **来源**：系统自动清空
- **说明**：跟卖不需要上传新图片

##### 🔟 Release date
- **规则**：根据运行时间判断
- **逻辑**：
  - 运行时间 < 15:00（北京时间）→ 使用昨天日期
  - 运行时间 ≥ 15:00（北京时间）→ 使用今天日期

##### 1️⃣1️⃣ launch date
- **规则**：与 Release date 相同
- **逻辑**：
  - 运行时间 < 15:00（北京时间）→ 使用昨天日期
  - 运行时间 ≥ 15:00（北京时间）→ 使用今天日期

---

#### 二、其他字段都不变，全部沿用老版本

除上述 11 个字段外，其他所有字段（如产品名称、描述、关键词等）均从老版本数据中复制，保持不变。

---

## 五、最终字段计算逻辑汇总（给开发）

```python
# 价格计算
new_price = old_price - 0.1
list_price = new_price + 10

# 日期计算（北京时间）
if run_time < 15:00:
    list_date = yesterday
    launch_date = yesterday
else:
    list_date = today
    launch_date = today

# 固定值
quantity = 0
image = null
product_id_type = "ASIN"
```

## 六、人工复核清单（上传前）

上传前必须确认：

- [ ] SKU 是否正确
- [ ] ASIN 是否匹配
- [ ] price 是否 -0.1
- [ ] list_price 是否 +10
- [ ] quantity 是否为 0
- [ ] image 是否清空
- [ ] 日期是否符合 3PM 规则

✅ 确认无误后上传 Amazon

## ⚠️ 现存问题（关键坑位）

### 问题描述

**BI 上新排查界面只显示：**
👉 已入库的部分 SKU

**但实际业务要求是：**
只要一个 SKC（颜色）开始入库，该颜色下所有 SKU（所有尺码）都必须安排跟卖上新

### 问题影响

**问题在于：**
- 可能只入库 2 个 SKU
- 但该颜色共有 8 个 SKU
- 如果只按 BI 显示的 2 个做跟卖
- **会造成同色断码、变体结构异常**

---

## 解决方式（新增必要步骤）

### 核心原则

**必须按照原有老版本有的全部 SKU，完成所有 SKU 的跟卖上新**

### 确保：
- ✅ 不漏尺码
- ✅ 不漏 SKU
- ✅ 不破坏变体结构
