# 加色加码系统问题分析

## 问题汇总

### 1. 后缀问题
**现象：**
- DAMA 店铺链接出来后缀还是 `-USA`，但实际应该是 `-PL` 和 `-PLPH`
- PZ 店铺链接出来后缀还是 `-USA`，但实际应该是 `-DA` 和 `-DAPH`

**数据验证：**
从 DA-0.xlsm 源文件看到：
```
SKU: EE00466BD14-PL       Parent SKU: EP00461-PL
SKU: EE00466BD14-PLPH     Parent SKU: EP00461-PLPH
```

从 PZ-0.xlsm 源文件看到：
```
SKU: EE00466BD14-PL       (应该是 -DA/-DAPH)
SKU: EE00466BD14-PLPH
```

**问题根源：**
`excel_processor.py:1885-1888` 中的后缀逻辑：
```python
final_suffix = info.suffix if info.suffix else (source_info.suffix if source_info else "")
if not final_suffix:
    final_suffix = "-USA"  # 默认值硬编码为 -USA
final_suffix = final_suffix.upper()
```

### 2. Parent SKU 字段对应问题
**现象：**
Parent SKU 字段对应不上

**当前逻辑：**
`excel_processor.py:1890` 和 `2056-2064`：
```python
parent_sku = f"{info.product_code}{final_suffix}"  # 行1890

# 行2056-2064：优先使用源文件的 parent_sku
source_parent_sku = read_source_value(source_header_map, source_row_values, ["parent sku"])
parent_sku_to_write = source_parent_sku if source_parent_sku else parent_sku
```

**问题：**
- 如果后缀错误（-USA），生成的 parent_sku 也会错误
- 源文件中的 parent_sku 可能也不正确（如 DA-0.xlsm 中 `EP00461-PL` 应该对应款号）

### 3. Variation Theme 字段问题
**现象：**
Variation Theme 应该是固定值 `SizeName-ColorName`

**当前逻辑：**
`excel_processor.py:1228-1237`：
```python
def normalize_variation_theme(value: Any, style_code: str, lookup_style_code: str, suffix: str) -> str:
    text = str(value).strip() if value is not None else ""
    upper_text = text.upper()
    if upper_text == "SIZE_NAME/COLOR_NAME (DEPRECATED: DO NOT USE)":
        return "SizeName-ColorName"
    if upper_text == "SIZE/COLOR":
        return "SizeColor"
    if not text:
        return f"{style_code}{resolve_first_suffix_for_style(lookup_style_code, suffix)}"
    return text
```

**问题：**
- 源文件中是 `SIZE/COLOR`，被转换为 `SizeColor`
- 应该统一转换为 `SizeName-ColorName`

### 4. Outer Material Type 字段问题
**现象：**
Outer Material Type 应该填写 `chiffon`, `sequin` 之类的材质

**当前状态：**
- 在 DAMA 输出模板中，字段存在：`outer_material_type1-5`（列14-18）
- 在 DA-0.xlsm 源文件中，对应字段是 `material`（列53-55）
- 当前代码中**没有处理 Outer Material Type 字段**

## 店铺配置对照表

| 店铺 | 模板类型 | 后缀（标准/Plus） | Image Variant | 源文件前缀 |
|------|----------|-------------------|---------------|------------|
| DAMA | DaMaUS   | -PL / -PLPH       | PL            | DA         |
| EP   | EPUS     | -USA / -PH        | L             | EP         |
| PZ   | PZUS     | -DA / -DAPH       | PL            | PZ         |

## 修复方案

### 方案1：后缀问题
**修改位置：** `excel_processor.py:1885-1888`

**建议：**
1. 根据模板类型（template_type）确定默认后缀
2. 检测 SKU 是否为 Plus 尺码（14+），决定是否添加 PH 后缀

```python
# 根据模板类型确定店铺后缀
store_suffix_map = {
    "DaMaUS": ("-PL", "-PLPH"),
    "EPUS": ("-USA", "-PH"),
    "PZUS": ("-DA", "-DAPH"),
}
base_suffix, plus_suffix = store_suffix_map.get(template_type, ("-USA", "-PH"))

# 判断是否为 Plus 尺码
is_plus_size = False
if final_size:
    try:
        size_num = int(re.search(r'\d+', str(final_size)).group())
        is_plus_size = size_num >= 14
    except:
        pass

final_suffix = plus_suffix if is_plus_size else base_suffix
```

### 方案2：Parent SKU 问题
**修改位置：** `excel_processor.py:1890` 和 `2056-2064`

**建议：**
Parent SKU 应该是 `{款号}{后缀}`，不包含颜色和尺码

```python
parent_sku = f"{info.product_code}{final_suffix}"
# 不再从源文件读取 parent_sku，直接使用生成的值
for col in parent_sku_cols:
    output_ws.cell(row=output_row_idx, column=col).value = parent_sku
```

### 方案3：Variation Theme 问题
**修改位置：** `excel_processor.py:1228-1237`

**建议：**
统一返回 `SizeName-ColorName`

```python
def normalize_variation_theme(value: Any, style_code: str, lookup_style_code: str, suffix: str) -> str:
    # 固定返回 SizeName-ColorName
    return "SizeName-ColorName"
```

### 方案4：Outer Material Type 问题
**修改位置：** `excel_processor.py` 新增字段处理逻辑

**建议：**
1. 从源文件的 `material` 字段读取材质信息
2. 写入到输出模板的 `outer_material_type1-5` 字段

```python
# 在字段复制逻辑中添加
material_cols = output_header_map.get("outer material type", [])
source_materials = []
for i in range(1, 6):
    mat = read_source_value(source_header_map, source_row_values, [f"material#{i}", "material"])
    if mat:
        source_materials.append(mat)

for idx, col in enumerate(material_cols):
    if idx < len(source_materials):
        output_ws.cell(row=output_row_idx, column=col).value = source_materials[idx]
```

## 需要确认的问题

1. **PZ 店铺的源文件后缀不对**：PZ-0.xlsm 中的 SKU 后缀是 `-PL/-PLPH`，但应该是 `-DA/-DAPH`。这是源文件错误还是配置错误？

2. **Plus 尺码判断逻辑**：是否所有 14+ 尺码都算 Plus？是否有例外？

3. **Material 字段映射**：源文件中的 `material` 字段是否就是要填入 `outer_material_type` 的内容？

4. **Parent SKU 生成规则**：是否所有店铺的 Parent SKU 都是 `{款号}{后缀}` 格式？
