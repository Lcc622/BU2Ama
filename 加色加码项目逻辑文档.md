## 📋 项目概述

### 目标
处理亚马逊产品数据，实现颜色和尺码的批量变更（加色加码）

### 数据源
- **亚马逊下载表格**：三张表格为同一数据源，需聚合处理
- **命名规则**：`店铺-0`, `店铺-1`, `店铺-2`
  - 示例：EP-0, EP-1, EP-2
- **映射数据源**：
  - EP系列表格 `Template sheet`
  - `All Listing Report`

### 输出表格需求
1. ✅ 输入和输出表格同步，只需修改 `Template sheet`
2. ✅ 所有字段必须填充完整，**黄色为必调字段**
3. ✅ **row3** 为输入表字段，**row2** 为输出表字段
4. ✅ 价格表格以 **SKU 作为唯一值**匹配

### 更换 SKC 逻辑
当颜色从 `BDG` 变更为 `TE` 时：
- 原图片：`ES0128BDG-L11.jpg`
- 新图片：`ES0128TE-L11.jpg`

**关键点**：只替换 SKU 中的颜色代码部分，保持变体后缀不变

## 🕐 Launch Date 计算规则

### 北京时间 3PM 规则

```
IF 当前北京时间 < 15:00 (3PM):
    Launch Date = 前一天 (T-1)
ELSE:
    Launch Date = 当天 (T)
```

### 示例

| 当前北京时间 | Launch Date |
|------------|-------------|
| 2026-02-26 14:59 | 2026-02-25 |
| 2026-02-26 15:00 | 2026-02-26 |
| 2026-02-26 15:01 | 2026-02-26 |

### 时区转换
- 北京时间 = UTC+8
- 系统需要将服务器时间转换为北京时间后再判断

## 🔄 SKU 格式规范

### SKU 结构
```
{产品代码}{颜色代码}{尺码}-{后缀}
```

### 组成部分

| 部分 | 长度 | 示例 | 说明 |
|-----|------|------|------|
| 产品代码 | 7-8 字符 | ES0128B, EG02088 | 产品唯一标识 |
| 颜色代码 | 2 字符 | BDG, TE, LV | 大写字母 |
| 尺码 | 2 数字 | 02, 14, 26 | 两位数字 |
| 后缀 | 可选 | -PH, -DA | 以 `-` 开头 |

### 示例
```
ES0128BDG02-PH
├─ ES0128B: 产品代码
├─ DG: 颜色代码 (Dark Green)
├─ 02: 尺码
└─ -PH: 后缀
```

### Parent SKU 规则
```
Parent SKU = {产品代码}{后缀}
```

示例：
- Child SKU: `ES0128BDG02-PH`
- Parent SKU: `ES0128B-PH`

## 📝 数据清洗规则

### 剔除与颜色不相关的部分
在产品描述和标题中，需要剔除非加色的颜色词，避免描述异常

**示例**：
```
原标题：Ever-Pretty Women's Dress Dark Green US02
新标题：Ever-Pretty Women's Dress Teal US02
```

**注意**：
- 只替换与 SKU 颜色相关的颜色词
- 保留品牌名称、产品类型等固定文本
- 确保语法正确，避免出现重复或缺失的颜色词

## 🔄 跨产品加色逻辑（重要）

### 什么是跨产品加色？

跨产品加色是指将一个产品的颜色变体加到另一个产品的 Parent SKU 下。

### 传统加色 vs 跨产品加色

#### 传统加色（同产品内）
```
原始：EG02088BK14-PH
新增：EG02088MG14-PH
Parent SKU：EG02088-PH（不变）
```

#### 跨产品加色（本案例）
```
原始产品：EG02088（后缀 -PH）
目标产品：EG02084（后缀 -USC）
操作：将 EG02088MG 加到 EG02084-USC 下

结果：
Child SKU：EG02088MG14-USC（待确认格式）
Parent SKU：EG02084-USC（必须改变）
```

### 跨产品加色的字段规则

| 字段类型 | 数据来源 | 示例 | 说明 |
|---------|---------|------|------|
| **Child SKU** | 原产品代码 + 新颜色 + 目标后缀 | EG02088MG14-USC | 保持原产品代码 |
| **Parent SKU** | 目标产品 Parent | EG02084-USC | ⚠️ 必须改为目标 Parent |
| **Product Name** | 目标产品样本数据 | 从 EG02084-USC 复制 | ⚠️ 不是原产品标题 |
| **Product Description** | 目标产品样本数据 | 从 EG02084-USC 复制 | ⚠️ 不是原产品描述 |
| **图片 URL** | 原产品代码 + 新颜色 | EG02088MG-L1.jpg | 保持原产品代码 |
| **价格** | 原产品价格 | 87.79 | 从原产品获取 |
| **尺码** | 原产品尺码 | 14 | 从原产品获取 |
| **Relationship Type** | 固定值 | Variation | 必填 |

### 示例：EG02088MG 加色到 EG02084-USC

#### 输入数据（EG02088-PH）
```
SKU: EG02088BK14-PH
Parent SKU: EG02088-PH
Product Name: Ever-Pretty Women's Spring Chiffon Semi Formal Dresses...
Price: 87.79
```

#### 输出数据（加色到 EG02084-USC）
```
SKU: EG02088MG14-USC
Parent SKU: EG02084-USC ← 改变
Product Name: Ever-Pretty Women's Spring Elegant Round Neck... ← 从 EG02084 复制
Price: 87.79 ← 保持原价格
Main Image: https://eppic.s3.amazonaws.com/EG02088MG-L1.jpg ← 原产品代码+新颜色
Relationship Type: Variation
```

### 跨产品加色的触发机制

**待确认**：系统如何识别跨产品加色？

可能的方案：
1. **前端指定**：用户在界面上选择目标 Parent SKU
2. **配置文件**：维护产品映射关系
3. **命名规则**：根据后缀变化自动推断

### 常见问题

#### Q1: Child SKU 的产品代码用哪个？
**A**: 使用原产品代码（EG02088），不是目标产品代码（EG02084）

#### Q2: 标题和描述从哪里获取？
**A**: 从目标产品（EG02084-USC）的样本 SKU 复制，不是从原产品复制

#### Q3: 图片 URL 用哪个产品代码？
**A**: 使用原产品代码 + 新颜色代码（EG02088MG）

#### Q4: 价格从哪里获取？
**A**: 从原产品（EG02088）的价格获取
