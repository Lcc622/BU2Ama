# 后缀标准化逻辑更新

> 更新日期：2026-02-26
>
> 更新内容：后缀标准化规则

---

## 📋 变更说明

### 新规则

**唯一值用 style 来截取，SKU 的前 7 位匹配 + 后缀**

**后缀标准化规则**：
- `-PH` → 保持 `-PH`
- `-USA` → 保持 `-USA`
- `-USB` → 统一为 `-USA`
- `-USC` → 统一为 `-USA`
- 无后缀 → 统一为 `-USA`

**最终输出**：
- 只生成两条数据：`-PH` 和 `-USA`

---

## 🔄 处理流程

### 输入阶段

```
输入 SKU:
1. EG02088BK14-PH  → style=EG02088, 后缀=-PH
2. EG02088BK14-USA → style=EG02088, 后缀=-USA
3. EG02088BK14-USC → style=EG02088, 后缀=-USA (标准化)
4. EG02088BK14-USB → style=EG02088, 后缀=-USA (标准化)
5. EG02088BK14     → style=EG02088, 后缀=-USA (标准化)
```

### 标准化阶段

```python
# 后缀标准化逻辑
if suffix in ["-USB", "-USC", ""]:
    normalized_suffix = "-USA"
else:
    normalized_suffix = suffix  # -PH 或 -USA 保持不变
```

### 输出阶段

```
输出 SKU (加色 MG):
1. EG02088MG14-PH  (1条)
2. EG02088MG14-USA (1条)

总计：2条数据
```

**文件**：`backend/app/core/excel_processor.py`

```python
# 行 232-245 (原代码)
if info and info.product_code in selected_prefixes:

# 修改后
if info and info.product_code in selected_prefixes:
    # 后缀标准化：USB/USC/无后缀 → USA
    original_suffix = info.suffix or ""
    if original_suffix in ["-USB", "-USC", ""]:
        normalized_suffix = "-USA"
    else:
        normalized_suffix = original_suffix

    if normalized_suffix not in all_data:
        all_data[normalized_suffix] = []
        sample_rows[normalized_suffix] = list(row)

    all_data[normalized_suffix].append(info)
```

## 📊 影响分析

### 数据量变化

**之前**：
- 可能生成 4 个 sheet：PH, USA, USB, USC
- 每个 style 可能有 4 条数据

**现在**：
- 只生成 2 个 sheet：PH, USA
- 每个 style 只有 2 条数据

**减少比例**：50%

### 优势

1. **简化输出**：只有两个后缀，更容易管理
2. **统一标准**：USB/USC 统一为 USA，避免混淆
3. **减少冗余**：不再为每个后缀生成单独的数据

### 注意事项

1. **样本行选择**：
   - 如果输入有 USB/USC，样本行会被标准化为 USA
   - 确保 USA 的样本行数据正确

2. **价格匹配**：
   - 价格仍然从原始 SKU 匹配
   - 不受后缀标准化影响

3. **图片 URL**：
   - 图片 URL 使用原始产品代码
   - 不受后缀标准化影响
