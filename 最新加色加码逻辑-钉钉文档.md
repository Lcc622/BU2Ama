## 🎯 加色加码板块清洗逻辑设置（通用）

### 核心原则

**加色加码逻辑一致**

### 🔑 唯一值匹配规则（最新更新）

**唯一值用 style 来截取，SKU 的前 7 位匹配 + 后缀**

**后缀处理规则**：
- `-PH` → 保持 `-PH`
- `-USA` → 保持 `-USA`
- `-USB` → 统一为 `-USA`
- `-USC` → 统一为 `-USA`
- 无后缀 → 统一为 `-USA`

**最终输出**：
- 只生成两条数据：`-PH` 和 `-USA`

**说明**：
- 使用 SKU 的**前 7 位**作为产品代码（style）
- 唯一值 = style（前7位）+ 标准化后缀
- 后缀标准化：USB/USC/无后缀 → USA
- 每个 style 只生成两个后缀的数据

**示例**：
```
输入 SKU:
- EG02088BK14-PH  → style=EG02088, 后缀=-PH
- EG02088BK14-USC → style=EG02088, 后缀=-USA (标准化)
- EG02088BK14-USB → style=EG02088, 后缀=-USA (标准化)
- EG02088BK14     → style=EG02088, 后缀=-USA (标准化)

输出 SKU (加色 MG):
- EG02088MG14-PH  (1条)
- EG02088MG14-USA (1条)

总计：2条数据
```

## 🎨 SKU 格式规范

### SKU 结构
```
{产品代码(7位)}{颜色代码(2位)}{尺码(2位)}-{后缀}
```

### 组成部分

| 部分 | 长度 | 示例 | 说明 |
|-----|------|------|------|
| 产品代码 (Style) | 7 位 | EG02088 | 产品唯一标识（前7位） |
| 颜色代码 | 2 位 | MG, BK, TE | 大写字母 |
| 尺码 | 2 位 | 14, 02, 26 | 两位数字 |
| 后缀 | 可选 | -PH, -USC | 以 `-` 开头 |

### 示例
```
EG02088MG14-USC
├─ EG02088: 产品代码 (前7位)
├─ MG: 颜色代码
├─ 14: 尺码
└─ -USC: 后缀
```

## 🔄 跨后缀加色逻辑

### 什么是跨后缀加色？

跨后缀加色是指将一个产品（某个后缀）的颜色变体加到另一个后缀的 Parent SKU 下。

### 匹配规则

**唯一值用 style 来截取，SKU 的前 7 位匹配**

**步骤**：
1. 从 SKU 中提取前 7 位作为 style（产品代码）
2. 根据 style 匹配目标后缀的产品
3. 生成新的 SKU（原 style + 新颜色 + 目标后缀）

### 示例：EG02088 从 -PH 加色到 -USC

#### 输入数据
```
SKU: EG02088BK14-PH
Style: EG02088 (前7位)
Parent SKU: EG02088-PH
```

#### 输出数据
```
SKU: EG02088MG14-USC
Style: EG02088 (前7位，保持不变)
Parent SKU: EG02084-USC (根据 style 匹配目标 Parent)
```

**关键点**：
- Style（前7位）用于匹配
- 新 SKU 使用原 style + 新颜色 + 目标后缀
- Parent SKU 根据 style 匹配目标后缀的 Parent

### 1. 更新 SKU 解析逻辑

```python
# 当前实现
SKU_PATTERN = re.compile(r'^([A-Z]{2}\d{5}[A-Z]?)([A-Z]{2})(\d{2})(-[A-Z]+)?$')

# 建议更新为
def parse_sku(sku):
    # 提取前7位作为 style
    style = sku[:7] if len(sku) >= 7 else None
    # 提取颜色代码（第8-9位）
    color_code = sku[7:9] if len(sku) >= 9 else None
    # 提取尺码（第10-11位）
    size = sku[9:11] if len(sku) >= 11 else None
    # 提取后缀
    suffix = sku[11:] if len(sku) > 11 else ''

    return {
        'style': style,
        'color_code': color_code,
        'size': size,
        'suffix': suffix
    }
```

### 2. 实现字段同步逻辑

```python
# 定义特殊处理字段
SPECIAL_FIELDS = [
    'seller_sku', 'style_number', 'parent_sku',
    'color', 'color_map',
    'main_image', 'other_image_1', 'other_image_2', ...,
    'price', 'launch_date'
]

# 定义保留字段（固定值）
PRESERVE_FIELDS = [
    'brand_name', 'product_type', 'update_delete',
    'target_gender', 'age_range',
    'size_system', 'size_class', 'body_type', 'height_type',
    'parentage', 'relationship_type', 'variation_theme'
]

def process_field(field_name, input_value, template_value):
    if field_name in SPECIAL_FIELDS:
        # 特殊处理
        return process_special_field(field_name, input_value)
    elif field_name in PRESERVE_FIELDS:
        # 保留字段（使用固定值或模板值）
        return template_value
    elif input_value:
        # 非特殊字段，有数据则同步
        return input_value
    else:
        # 无数据则使用模板值
        return template_value
```
