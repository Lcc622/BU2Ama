## 🔄 跨后缀加色逻辑（核心）

### 什么是跨后缀加色？

跨后缀加色是指将一个产品（某个后缀）的颜色变体加到另一个后缀的 Parent SKU 下。

### 后缀类型

系统支持以下后缀：
- `-PH`
- `-USA`
- `-USB`
- `-USC`

### 匹配规则

**唯一值用 style 来截取**：
- 产品代码（style）= SKU 去掉颜色、尺码、后缀后的部分
- 例如：`EG02088BK14-PH` → style = `EG02088`

### 跨后缀加色示例

#### 场景：EG02088 从 -PH 加色到 -USC

**原始产品**：
```
SKU: EG02088BK14-PH
Style: EG02088
Parent SKU: EG02088-PH
```

**目标产品**：
```
Style: EG02084 (或 EG02088，取决于业务规则)
Parent SKU: EG02084-USC (或 EG02088-USC)
```

**生成结果**：
```
SKU: EG02088MG14-USC
Style: EG02088
Parent SKU: EG02084-USC (根据 style 匹配目标 Parent)
```

### 字段数据来源规则

| 字段类型 | 数据来源 | 说明 |
|---------|---------|------|
| **Child SKU** | 原产品 style + 新颜色 + 目标后缀 | EG02088MG14-USC |
| **Parent SKU** | 目标产品 Parent（根据 style 匹配） | EG02084-USC |
| **Product Name** | 目标产品样本数据 | 从目标 Parent 的样本 SKU 复制 |
| **Product Description** | 目标产品样本数据 | 从目标 Parent 的样本 SKU 复制 |
| **图片 URL** | 原产品 style + 新颜色 | EG02088MG-L1.jpg |
| **价格** | 原产品价格（从 All Listing Report） | 87.79 |
| **尺码** | 原产品尺码 | 14 |
| **Relationship Type** | 固定值 | Variation |

## 📝 关键字段说明

### 1. SKU 格式
```
{产品代码}{颜色代码}{尺码}-{后缀}
```

**示例**：
- `EG02088BK14-PH`
- `EG02088MG14-USC`

### 2. Style（产品代码）
```
{产品代码}
```

**示例**：
- `EG02088`
- `EG02084`

**作用**：
- 作为唯一标识符匹配不同后缀的产品
- 用于跨后缀加色时查找目标 Parent SKU

### 3. Parent SKU 格式
```
{产品代码}-{后缀}
```

**示例**：
- `EG02088-PH`
- `EG02084-USC`

### 4. 价格匹配
- **数据源**：`All Listing Report`
- **匹配键**：SKU（唯一值）
- **字段**：`standard_price`

**关键点**：图片 URL 使用原产品的 style + 新颜色代码

新颜色：MG
目标后缀：-USC

- https://eppic.s3.amazonaws.com/EG02088MG-L2.jpg
- ...
```

**不是**：
```

## 🚀 处理流程

### 1. 数据输入
```
用户上传 → EP-0.xlsm, EP-1.xlsm, EP-2.xlsm
```

### 2. 数据聚合
```
读取所有文件 → 合并 SKU 数据 → 按 style 分组
```

### 3. SKU 解析
```
EG02088BK14-PH → {
  style: "EG02088",
  colorCode: "BK",
  size: "14",
  suffix: "-PH"
}
```

### 4. 跨后缀匹配
```
根据 style (EG02088) 查找目标后缀 (-USC) 的 Parent SKU
→ 找到 EG02084-USC（或 EG02088-USC）
```

### 5. 字段生成
```
- Child SKU: EG02088MG14-USC
- Parent SKU: EG02084-USC
- Product Name: 从 EG02084-USC 的样本复制
- Price: 从 EG02088 的原价格获取 (87.79)
- Images: EG02088MG-L1.jpg, EG02088MG-L2.jpg, ...
- Relationship Type: Variation
```

### 6. 数据输出
```
生成 Excel → 按后缀分 Sheet → 下载
```

## ⚠️ 常见问题和注意事项

### 问题 1：标题不一致
**现象**：输出的标题与原产品不匹配

**原因**：跨后缀加色时，标题应该从目标 Parent 的样本数据复制，而不是从原产品复制

**解决**：根据 style 匹配目标 Parent，从目标 Parent 的样本 SKU 复制标题

### 问题 2：价格不一致
**现象**：输出的价格与原产品不匹配

**原因**：价格应该从原产品的 `All Listing Report` 获取

**解决**：使用原产品的 SKU 从 `All Listing Report` 匹配价格

### 问题 3：图片 URL 错误
**现象**：图片 URL 指向错误的产品代码

**原因**：图片 URL 应该使用原产品的 style + 新颜色代码

**解决**：确保图片 URL 格式为 `{原产品style}{新颜色}-{变体}.jpg`

### 问题 4：Parent SKU 错误
**现象**：Parent SKU 仍然是原产品的 Parent

**原因**：跨后缀加色时，Parent SKU 必须改为目标后缀的 Parent

**解决**：根据 style 匹配目标后缀的 Parent SKU

### 问题 5：Relationship Type 缺失
**现象**：Relationship Type 字段为空

**原因**：该字段是必填字段

**解决**：确保填充 `Variation`
