## 📋 问题概述

在处理 **EG02088MG 加色到 EG02084-USC** 的案例中，发现输出结果存在多个字段错误。

### 测试案例
- **原始产品**：EG02088（后缀 -PH）
- **目标产品**：EG02084（后缀 -USC）
- **操作**：将 EG02088 的 MG 颜色加到 EG02084-USC 这个 Parent 下

## 🔴 发现的问题

### 1. item_name 标题不一致

| 字段 | 期望值 | 实际输出 |
|-----|--------|---------|
| **item_name** | Ever-Pretty Women's Spring Chiffon Semi Formal Dresses V Neck Ruffle Sleeves High Low Cocktail Dress for Wedding Guest | Ever-Pretty Women's Spring Elegant Round Neck Short Sleeve Chiffon Empire Waist Floor Length Formal Dresses Bridesmaid Dress |

**问题原因**：
- 系统从错误的产品代码（EG02084）复制了标题
- 应该保持 EG02088 的原始标题，只替换颜色词

---

### 2. standard_price 价格不一致

| 字段 | 期望值 | 实际输出 |
|-----|--------|---------|
| **standard_price** | 87.79 | 89.99 |

**问题原因**：
- 价格匹配逻辑可能从错误的 SKU 获取了价格
- 应该从 EG02088 的原始价格获取（87.79）

---

### 3. 图片输出不对

**问题描述**：
- 图片 URL 指向了错误的产品代码或颜色代码

**期望格式**：
```
https://eppic.s3.amazonaws.com/EG02088MG-L1.jpg
https://eppic.s3.amazonaws.com/EG02088MG-L2.jpg
https://eppic.s3.amazonaws.com/EG02088MG-L3.jpg
...
```

**问题原因**：
- 图片 URL 生成逻辑可能使用了错误的产品代码
- 应该使用 **EG02088 + MG**（原产品代码 + 新颜色代码）

---

### 4. Parent SKU 不对

| 字段 | 期望值 | 实际输出 |
|-----|--------|---------|
| **Parent SKU** | EG02084-USC | EG02088-PH |

**问题原因**：
- 系统保持了原始的 Parent SKU（EG02088-PH）
- **关键错误**：跨产品加色时，Parent SKU 必须改为目标产品的 Parent

**正确逻辑**：
- EG02088MG 加色到 EG02084-USC 上
- 所以 Parent SKU = `EG02084-USC`

## 🎯 核心问题：跨产品加色逻辑

### 什么是跨产品加色？

**传统加色**（同产品内）：
```
EG02088BK14-PH → EG02088MG14-PH
Parent: EG02088-PH → EG02088-PH（不变）
```

**跨产品加色**（本案例）：
```
EG02088BK14-PH → EG02088MG14-USC（或 EG02084MG14-USC？）
Parent: EG02088-PH → EG02084-USC（必须改变）
```

### 跨产品加色的特点

1. **Child SKU**：保持原产品代码，但改变后缀
   - 格式待确认：`EG02088MG14-USC` 还是 `EG02084MG14-USC`？

2. **Parent SKU**：必须改为目标产品的 Parent
   - `EG02084-USC`

3. **Product Name, Description**：从目标产品（EG02084-USC）的样本数据复制
   - 不是从原产品（EG02088）复制

4. **图片 URL**：使用原产品代码 + 新颜色代码
   - `EG02088MG-L1.jpg`（不是 EG02084MG）

5. **价格**：从原产品（EG02088）获取
   - 87.79

## 🛠️ 可能的解决方案

### 方案 1：前端指定目标 Parent
用户在界面上选择：
1. 上传原始文件（EG02088）
2. 选择要加的颜色（MG）
3. **指定目标 Parent SKU**（EG02084-USC）
4. 系统根据目标 Parent 查找样本数据

### 方案 2：配置文件映射
创建一个映射配置：
```json
{
  "EG02088-PH": {
    "targetParent": "EG02084-USC",
    "sampleSKU": "EG02084BK14-USC"
  }
}
```

### 方案 3：命名规则推断
根据文件名或 SKU 格式自动推断：
- 如果后缀从 `-PH` 变为 `-USC`，则查找对应的目标 Parent
